## 二、关键机制解析

### 1. 数据注入与接收

**核心策略：高频数据的异步处理**

```python
# 老项目关键设计（app.py）
@app.route("/api/bluetooth-data", methods=["POST"])
def receive_bluetooth_data():
    # 1. 数据接收与标准化
    body = request.get_json()

    # 3. 触发算法计算（非阻塞）
    try:
        # 仅当数据量足够时触发计算
        if len(fatigue_signal_buffer) >= FATIGUE_MIN_SAMPLES:
            fatigue_output = run_fatigue_pipeline(raw_signal, ...)
            broadcast_fatigue(fatigue_output)
    except Exception as e:

```

**经验总结：**
- **数据标准化**：统一字段格式，避免上游数据格式差异影响下游
- **异步处理**：数据接收与算法计算解耦，避免阻塞 HTTP 响应
- **容错设计**：算法计算失败不影响数据接收，保证系统稳定性

### 2. 缓冲区管理 (Buffer)

**双层缓冲架构**

```python
# 老项目关键设计
FATIGUE_WINDOW_SECONDS = 10  # 计算窗口：最近10秒
SAMPLING_RATE = 100          # 采样率
FATIGUE_MIN_SAMPLES = SAMPLING_RATE * 3  # 最少3秒数据启动计算

# 缓冲区1：原始蓝牙数据（保留最近1000条）
bluetooth_data = []

# 缓冲区2：信号数据（滑动窗口，保留最近10秒）
fatigue_signal_buffer = []

def add_signal_data(chunk):
    fatigue_signal_buffer.extend(chunk)
    max_len = int(FATIGUE_WINDOW_SECONDS * SAMPLING_RATE)
    if len(fatigue_signal_buffer) > max_len:
        fatigue_signal_buffer = fatigue_signal_buffer[-max_len:]  # 滑动窗口
```

**关键设计点：**

| 机制 | 作用 | 实现方式 |
|-----|------|---------|
| **滑动窗口** | 保证算法使用最新数据 | 切片操作 `buffer[-max_len:]` |
| **双缓冲分离** | 原始数据与计算数据解耦 | 两个独立缓冲区 |
| **容量限制** | 防止内存溢出 | 固定最大长度，自动裁剪 |
| **阈值触发** | 避免无效计算 | 数据量达标才触发算法 |

### 3. 实时推送 (WebSocket)

**广播机制设计**

```python
# 老项目关键设计
ws_clients = set()  # 存储所有连接的客户端

def broadcast(payload: dict):
    dead = []
    for client in list(ws_clients):
        ok = safe_ws_send(client, payload)
        if not ok:
            dead.append(client)  # 标记失效连接
    for c in dead:
        ws_clients.discard(c)  # 清理失效连接

def safe_ws_send(ws, payload: dict) -> bool:
    try:
        ws.send(json.dumps(payload))
        return True
    except Exception:
        return False  # 静默失败，由广播层统一清理
```

**心跳与重连机制**

```javascript
// 前端（App.tsx）关键设计
useEffect(() => {
    const ws = new WebSocket(WS_URL);

    // 心跳发送（15秒间隔）
    const pingTimer = setInterval(() => {
        try {
            ws.send(JSON.stringify({ type: "ping", ts: Date.now() }));
        } catch {}
    }, 15000);

    // 自动重连
    ws.onclose = () => {
        clearInterval(pingTimer);
        if (isMonitoring) {
            // 延迟重连，避免频繁重试
            setTimeout(() => connectWebSocket(), 3000);
        }
    };

    return () => {
        ws.close();
        clearInterval(pingTimer);
    };
}, [isMonitoring]);
```

**经验总结：**
- **客户端管理**：使用 `set` 存储连接，便于快速增删
- **优雅降级**：单客户端发送失败不影响其他客户端
- **心跳机制**：15秒间隔发送 ping，防止中间设备断开连接
- **自动重连**：断开后延迟重连，避免网络抖动导致频繁重试

### 3. 关键伪代码

#### 后端缓冲区管理

```python
class DataBuffer:
    def __init__(self, max_size: int = 1000, window_seconds: int = 10, sampling_rate: int = 100):
        self.raw_buffer = []  # 原始数据缓冲区
        self.signal_buffer = []  # 信号缓冲区（用于计算）
        self.max_size = max_size
        self.window_size = window_seconds * sampling_rate
        self.min_samples = sampling_rate * 3  # 最少3秒数据

    def add_data(self, data_point: dict):
        """添加数据点"""
        # 原始数据缓冲
        self.raw_buffer.append(data_point)
        if len(self.raw_buffer) > self.max_size:
            self.raw_buffer = self.raw_buffer[-self.max_size:]

        # 信号数据缓冲（滑动窗口）
        signal_data = data_point.get("rawData", [])
        self.signal_buffer.extend(signal_data)
        if len(self.signal_buffer) > self.window_size:
            self.signal_buffer = self.signal_buffer[-self.window_size:]

    def can_compute(self) -> bool:
        """是否有足够数据进行计算"""
        return len(self.signal_buffer) >= self.min_samples

    def get_signal_array(self) -> np.ndarray:
        """获取信号数据数组"""
        return np.asarray(self.signal_buffer, dtype=float)
```

#### 前端 WebSocket Hook

```typescript
export function useWebSocket<T>(url: string, onMessage: (data: T) => void) {
    const [status, setStatus] = useState<"disconnected" | "connecting" | "connected" | "error">("disconnected");

    useEffect(() => {
        if (!url) return;

        setStatus("connecting");
        const ws = new WebSocket(url);

        ws.onopen = () => {
            setStatus("connected");
            // 心跳
            const pingTimer = setInterval(() => {
                try {
                    ws.send(JSON.stringify({ type: "ping", ts: Date.now() }));
                } catch {}
            }, 15000);

            ws.onclose = () => {
                clearInterval(pingTimer);
                setStatus("disconnected");
            };
        };

        ws.onerror = () => setStatus("error");

        ws.onmessage = (event) => {
            try {
                const msg = JSON.parse(event.data);
                onMessage(msg as T);
            } catch {}
        };

        return () => ws.close();
    }, [url, onMessage]);

    return { status };
}
```

## 四、避坑指南

### 1. 数据量不足导致计算异常

**问题**：算法启动时数据不足，导致计算结果不准确或报错

**解决方案**：

```python
# 设置最小数据量阈值
MIN_SAMPLES = 300  # 3秒 * 100Hz

def compute_result(buffer):
    if len(buffer) < MIN_SAMPLES:
        return None  # 或返回默认值，不触发推送

    # 执行计算
    return run_algorithm(buffer)
```

### 2. 缓冲区内存溢出

**问题**：持续接收数据但不清理，导致内存占用持续增长

**解决方案**：

```python
# 方案1：固定大小裁剪
MAX_BUFFER_SIZE = 1000
buffer.append(data)
if len(buffer) > MAX_BUFFER_SIZE:
    buffer = buffer[-MAX_BUFFER_SIZE:]

# 方案2：定时清理（配合固定大小）
import threading
def clean_buffer():
    global buffer
    buffer = buffer[-MAX_BUFFER_SIZE:]
    threading.Timer(60, clean_buffer).start()  # 每分钟检查一次
```

### 3. WebSocket 连接泄漏

**问题**：客户端断开后，服务端仍保留无效连接，导致资源浪费

**解决方案**：

```python
# 使用安全发送函数，自动检测失效连接
def broadcast(payload):
    dead_clients = []
    for client in list(ws_clients):
        try:
            client.send(json.dumps(payload))
        except Exception:
            dead_clients.append(client)

    # 清理失效连接
    for client in dead_clients:
        ws_clients.discard(client)
```

### 4. 前端状态覆盖问题

**问题**：高频数据推送导致状态更新冲突，UI 显示异常

**解决方案**：

```typescript
// 使用函数式更新，确保基于最新状态
setCurrentData(prev => {
    const next = processData(newData, prev);
    return next;
});

// 合并更新而非替换
setHistoryData(prev => {
    const updated = [...prev, newPoint];
    return updated.slice(-MAX_HISTORY_LENGTH);  // 限制长度
});
```

### 5. 网络抖动导致数据乱序

**问题**：WebSocket 消息可能因网络原因乱序到达

**解决方案**：

```python
# 服务端：添加时间戳，确保有序
payload = {
    "type": "result",
    "data": result,
    "timestamp": int(time.time() * 1000)
}

# 前端：根据时间戳过滤旧数据
ws.onmessage = (event) => {
    const msg = JSON.parse(event.data);
    if (msg.timestamp <= lastReceivedTimestamp) {
        return;  // 跳过旧数据
    }
    lastReceivedTimestamp = msg.timestamp;
    updateState(msg.data);
};
```

## 五、总结

| 维度 | 老项目经验 | 新项目建议 |
